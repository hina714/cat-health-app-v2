# アプリ仕様書

## 目的

家族で共有できる猫の健康管理アプリ。
対象は自分の家族のみ（複数家族の管理は対象外）。

---

## 機能一覧

| 機能 | 状態 |
|---|---|
| アカウント登録 | ✅ 完成 |
| ログイン | ✅ 完成 |
| 猫の登録 | 🚧 未実装 |
| 記録管理 | 🚧 未実装 |

---

## 機能詳細

### 猫の登録

飼っている猫のプロフィールを登録する。

| 項目 | 種類 | 必須 |
|---|---|---|
| アイコン写真 | ファイル選択 | |
| 名前 | テキスト入力 | ✓ |
| 品種 | テキスト入力 | |
| 誕生日 | 日付入力 | |
| 去勢・避妊 | ラジオボタン（あり / なし / 不明） | |

---

### 記録管理

猫の日々の健康状態を記録・確認する。

#### 記録項目

| 項目 | 種類 |
|---|---|
| 体重 | 数値入力（kg） |
| 食事量 | 選択（よく食べた / 普通 / 少ない / 食べなかった） |
| 排泄 | 選択（正常 / 異常 / なし） |
| 体調 | 選択（良好 / 普通 / 不調） |
| 様子・メモ | テキスト入力（自由記述） |

#### 権限

- 記録には**誰が書いたか**（ユーザー名）を表示する
- 記録の編集は**書いた本人のみ**可能
- 家族は記録に対して**コメント**を投稿できる

---

## データ設計

```
users（ユーザー）
  └── records（記録）       ← 書いたユーザーに紐づく
  └── comments（コメント）  ← 書いたユーザーに紐づく

cats（猫）
  └── records（記録）       ← 対象の猫に紐づく
```

> **補足：猫は誰に紐づけるか**
>
> 家族全員が同じ猫のデータを共有するため、猫を特定の1ユーザーに紐づけると他の家族が閲覧・編集できなくなる。
> 今回は「1家族 = 1アプリ」の前提のため、複数家族を分ける仕組み（householdsテーブル等）は作らない。
> 代わりに、**全ユーザーが全ての猫を共有できる**シンプルな設計とする。
